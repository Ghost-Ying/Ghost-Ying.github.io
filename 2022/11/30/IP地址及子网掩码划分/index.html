IP地址及子网掩码划分

发表于 2022-11-30 分类于 思科CCNA网络基础入门 阅读次数：

一：IP（internet protocol）

1.1 概述

IP指网络互联协议，是TCP/IP体系中的网络层协议。

设计IP的目的是提高网络的可扩展性：

  • 一是解决互联网问题，实现大规模、异构网络的互联互通；
  • 二是分割顶层网络应用和底层网络技术之间的耦合关系，以利于双方独立发展；
  • 根据端到端的设计原则，IP只为主机提供了一种无连接、不可靠的、尽力而为的数据包传输服务。

例如：192.168.1.1，该IP采用的是点分十进制的表示方法。但在计算机的世界中采用的是点分二进制的方法，对应的地址表示：1111 1111.0000 0000.0000 0000.0000 0000。由此，通过运算我们可以得出IP地址一共有232次方个，约42.9亿个。

1.2 IP 地址分类

（1）IP划分

  • A类地址：1.0.0.1~126.255.255.254
  • B类地址：128.0.0.1~191.255.255.254
  • C类地址：192.0.0.1~223.225.225.254
  • D类地址：224.0.0.1~239.255.255.254（多播地址）
  • E类地址：240.0.0.1~255.255.255.254（科研预留）

（2）用途划分

  • 网络地址（network）：也可以称网络号，唯一指定了每个网络。同一个网络中的每台计算机都共享相同的网络地址，并用它作为自己的IP地址一部分。它定义了IP地址所属的网段。
  • 结点地址（host）：也可以称为主机地址，是在一个网络中用来标识每台计算机的，它也是唯一的标识符。相对于网络而言，它是用来独立标志指定计算机的。

1.3 私有IP地址

  • A类地址中：10.0.0.0到10.255.255.255
  • B类地址中：172.16.0.0到172.31.255.255
  • C类地址中：192.168.0.0到192.168.255.255

注意：私有IP地址一般用于公司内部，私有地址可以相同。

1.4 特殊的IP地址

  • 本地回环（loopback ）：127.0.0.1，通常使用ping命令来检测，网卡是否可用正常使用（测试地址）。
  • 广播地址：255.255.255.255
  • IP地址 0.0.0.0：代表任何网络
  • 向网上所有的主机发送报文，也就是说，不管物理网络特性如何，internet网支持广播传输。如136.78.255.255就是B类地址中的一个广播地址，你将信息送到此地址，就是将信息送给网络号为136.78的所有主机。

1.5 IP 地址分配主机的数量

根据子网掩码的位数，来计算IP地址的数量，即主机地址的个数。

  • A类地址：224个地址，约1677万个
  • B类地址：216个地址，65534个
  • C类地址：28个地址，254个

二：子网掩码

2.1 常见掩码

  • A类地址：1.0.0.1~126.255.255.254
    • 默认掩码：255.0.0.0 ==/8
  • B类地址：128.0.0.1~191.255.255.254
    • 默认掩码：255.255.0.0 ==/16
  • C类地址：192.0.0.1~223.225.225.254
    • 默认掩码：255.255.255.0 ==/24
  • 采用点分二进制表达式：0表示主机位，1表示网络位

2.2 VLSM

变长子网掩码（Variable length subnet masks）的出现是打破传统的以类（class）为标准的地址划分的方法，目的是缓解IP地址紧缺而产生的。

作用：节约IP地址空间。

注意事项：使用 VLSM 时，所采用的路由协议必须都能够支持它，这些路由协议包括RIPv2，OSPF，EIGPR 和 BGP。

子网划分

举例说明：某公司现有一C类网段 192.168.1.0/24

目前有以下部门：销售部 59台、技术部 27台、业务部 121台、会计部 10台。

需求：将一个C类网段合理分配给如下几个部门，保证地址合理即可。

  • 4个网络
    • 192.168.1.0~63
    • 192.168.1.64~127
    • 192.168.1.128~191
    • 192.168.1.192~255
    • 当划分到业务部时，发现地址不够，所以子网划分时，优先考虑主机数最大的网段。
  • 2个网络
    • 192.168.1.0 1-126 可容纳126台主机
    • 192.168.1.128 129-254 可容纳126台主机
    • 以此在网下划分即可；

当我们不断将有效位置1时，极限抵达192.168.1.0/30，此时对应的子网掩码为：255.255.255.252，此时仅剩2个主机位，只能实现1对1的端到端通信。

2.3 CIDR

CIDR（classless inter-Domain routing，无类域间路由选择），它消除了传统的A类、B类和C类地址，以及子网划分的概念，可以将原本划分的网络进行合并。使用一种无类别的域际路由选择算法，使它们合并成一条路由从而较少路由表中的路由条目减轻internet路由器的负担。
